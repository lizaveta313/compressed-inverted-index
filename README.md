# Сжатый инвертированный индекс

Учебный проект по теме: «Разработка и экспериментальное исследование сжатого
инвертированного индекса с использованием дельта-кодирования и
гамма-кодирования Элиаса».

## Краткое описание

Инвертированный индекс хранит для каждого слова список документов, где это
слово встречается. В сжатом варианте вместо самих `doc_id` хранятся разности
между соседними значениями, а эти разности кодируются гамма-кодом Элиаса.

Так индекс может занимать меньше места. Минус в том, что при поиске сжатые
списки нужно декодировать.

## Что реализовано

- простой токенизатор для русского и английского текста;
- обычный инвертированный индекс;
- дельта-кодирование списков `doc_id`;
- гамма-кодирование Элиаса;
- сжатый инвертированный индекс;
- небольшой эксперимент и тесты.

## Установка

Сначала лучше создать виртуальное окружение.

Для Windows:

```bat
python -m venv .venv
.venv\Scripts\activate
```

Для macOS/Linux:

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Установка зависимостей:

```bash
pip install -r requirements.txt
```

## Запуск тестов

```bash
pytest
```

## Запуск эксперимента

```bash
python -m compressed_inverted_index.experiment
```

Эксперимент:

- генерирует набор документов;
- строит обычный инвертированный индекс;
- строит сжатый индекс;
- сравнивает размер хранения;
- сравнивает время построения;
- проверяет поиск по запросам `ректор СПбГУ`, `ректор МГУ` и `university rector`.

## Краткий вывод

Дельта-кодирование делает списки документов меньше, потому что вместо больших
`doc_id` часто получаются небольшие разности. Гамма-кодирование Элиаса
компактно записывает положительные числа. В этом эксперименте сжатый индекс
занимает меньше места, но поиск по нему медленнее из-за декодирования.
