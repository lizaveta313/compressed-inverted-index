# Сжатый инвертированный индекс

Учебный проект по теме: «Разработка и экспериментальное исследование сжатого инвертированного индекса с использованием дельта-кодирования и гамма-кодирования Элиаса».

## Краткое описание

Инвертированный индекс хранит для каждого термина список документов, где этот термин встречается. В сжатом варианте postings lists сначала переводятся в дельты между соседними `doc_id`, а затем кодируются гамма-кодом Элиаса.

## Сделано

В проекте:

- реализован обычный инвертированный индекс;
- реализовано сжатие postings lists через дельта-кодирование и гамма-кодирование Элиаса;
- проведено сравнение обычного и сжатого индекса;
- проверяется поиск по запросам «Ректор СПбГУ» и «Ректор МГУ»;
- поддерживается загрузка данных из JSONL, полученных на втором модуле;
- формируются CSV и Markdown-отчёт по эксперименту.

```bash

```

```bash

```

## Формат данных из второго модуля

JSONL-файл: одна строка — один документ. Текст берётся из поля `text`, `content` или `body`.

```json
{"url": "https://spbu.ru/...", "text": "текст страницы"}
{"id": 123, "content": "текст публикации"}
```

## Запуск на данных второго модуля

```bash
python -m compressed_inverted_index.benchmark --data data/module2/spbu_documents.jsonl
```

С нужными размерами корпуса:

```bash
python -m compressed_inverted_index.benchmark --data data/module2/spbu_documents.jsonl --sizes 10 25 50
```

Если файла нет или в нём нет текстовых документов, программа выводит понятное сообщение без traceback.

## Запуск без реальных данных

```bash
python -m compressed_inverted_index.benchmark
```

В этом режиме используются сгенерированные документы. Они нужны как fallback, если данных из второго модуля нет.

## Запуск базового эксперимента

python -m compressed_inverted_index.experiment

## Откуда берутся 40 000 документов

Для проверки до 40 000 документов используется функция `generate_documents(count, seed=42)` из `compressed_inverted_index/data_loader.py`. Она не скачивает данные из интернета и не хранит отдельный файл на диске, а воспроизводимо генерирует синтетический корпус в памяти:

- берётся фиксированный словарь русских и английских терминов;
- для каждого документа случайно выбираются слова;
- по правилу добавляются фразы «Ректор СПбГУ», «Ректор МГУ» и `university rector`;
- `seed=42` делает результат повторяемым при каждом запуске.
